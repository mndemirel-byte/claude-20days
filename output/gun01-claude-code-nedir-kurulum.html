Claude Code Nedir ve Kurulum

“Terminalindeki AI takım arkadaşınla tanış”

Bu derste Claude Code'un ne olduğunu, nasıl çalıştığını ve diğer AI kodlama araçlarından farkını öğreneceksin. Dersin sonunda Claude Code'u bilgisayarına kurup ilk komutunu çalıştıracak ve ilk kodunu yazdıracaksın.

BÖLÜM 1TEORİK TEMEL

1.1 Claude Code Nedir?

Claude Code, Anthropic tarafından geliştirilen agentic (ajan tabanlı) bir kodlama aracıdır. Terminalinde yaşayan; projeni anlayan ve doğal dil komutlarıyla kod yazabilen, dosya düzenleyen, komut çalıştıran ve git işlemlerini yöneten bir AI asistanıdır.

🔑 ÖnemliClaude Code bir chatbot değildir. Kopyala-yapıştır ile çalışmaz. Doğrudan senin proje klasöründe çalışır: dosyalarını okur, düzenleme yapar, terminal komutları çalıştırır ve git ile entegre olur. Bir 'AI takım arkadaşı' gibi düşün.

Claude Code ne yapabilir?

  • Kod yazma: Doğal dil ile tarif et, Claude üretsin ('Python ile bir REST API yaz').
  • Kod düzenleme: Mevcut dosyaları okuyup değişiklik önerir ve uygular.
  • Bug düzeltme: Hata mesajını yapıştır, Claude izleyip düzeltsin.
  • Test yazma: 'Auth modülü için testleri yaz, çalıştır, fail edenleri düzelt'.
  • Git işlemleri: Commit, branch, merge, PR işlemlerini doğal dil ile yap.
  • Refactoring & dokümantasyon: README, API docs, changelog oluşturma.

1.2 Nasıl Çalışır? (Agentic Loop)

Claude Code'un en önemli özelliği agentic çalışmasıdır. Bir görev verdiğinde sadece cevap dönmez; bir döngü içinde birden fazla adım atar:

AdımNe yaparÖrnek
1. AnlamaGörevi analiz eder, projeyi tararProje yapısını ve dosyaları inceler
2. PlanlamaYaklaşım belirler'Önce DB, sonra API, sonra test' planlar
3. Araç kullanımıDosya okur/yazar, komut çalıştırırDosya oluşturur, pip install çalıştırır
4. DoğrulamaSonucu kontrol ederTesti çalıştırır, hataları görür
5. İterasyonHata varsa düzeltirTest fail olursa kodu günceller
💡 İpucuBu döngü, Claude Code'u chatbot'lardan ayıran özelliktir. Bir chatbot'a 'test yaz' dersen kodu gösterir ama çalıştırmaz. Claude Code yazar, çalıştırır, fail edenleri düzeltir ve tekrar çalıştırır.

1.5 Modeller ve Effort Seviyeleri

Claude Code, model ailesini kısa alias'larla sunar. /model ile seçersin:

AliasModelEn iyi olduğu iş
opusClaude Opus 4.8Karmaşık akıl yürütme, mimari kararlar
sonnetClaude Sonnet 5Günlük kodlama (native 1M context)
haikuClaude Haiku 4.5Basit, hızlı görevler
fableClaude Fable 5En yetenekli; uzun/otonom görevler (varsayılan değil)
opusplanOpus + SonnetPlanlamada opus, uygulamada sonnet

Effort (çaba) seviyeleri: low, medium, high (varsayılan), xhigh, max. Daha yüksek effort = daha derin düşünme, daha çok token. Prompt içinde ultrathink yazarak tek seferlik derin düşünme tetikleyebilirsin. Fast mode ise Opus'ta daha hızlı yanıt verir.

🔑 ÖnemliContext window artık 200K değil: Sonnet 5 native 1M (1 milyon) token ile çalışır; Opus üst planlarda 1M'e otomatik yükselir. Bu, büyük kod tabanlarıyla çalışmayı kolaylaştırır.

2.2 Claude Code Kurulumu

Yöntem 1: Native Installer (önerilen) — ek bağımlılık gerektirmez, otomatik güncellenir.

# macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | bash
# Windows (PowerShell)
irm https://claude.ai/install.ps1 | iex
# Homebrew (macOS)
brew install --cask claude-code

# WinGet (Windows)
winget install Anthropic.ClaudeCode
⚠️ DikkatHomebrew/WinGet kurulumlarında otomatik güncelleme yoktur; manuel brew upgrade / winget upgrade gerekir. Native installer ise arka planda otomatik güncellenir.

Yöntem 2: npm ile kurulum — yalnızca Node.js 18+ mevcutsa tercih et.

node --version   # v18.0.0+ olmalı
npm install -g @anthropic-ai/claude-code
⚠️ DikkatAsla sudo npm install -g kullanma! İzin ve güvenlik sorunlarına yol açar. İzin hatası alırsan nvm kullan.

2.3 Kurulumu Doğrulama

claude --version    # sürüm numarası görmelisin
claude doctor       # kurulum tipi, sürüm, bağlantı kontrolü

2.4 İlk Çalıştırma ve Kimlik Doğrulama

  1. Terminal'de herhangi bir klasörde claude yaz ve çalıştır.
  2. Tarayıcın açılır; Anthropic hesabınla giriş yap.
  3. Giriş başarılıysa terminal'e dön — Claude Code hazır.
  4. İlk komutunu çalıştır: /status (model, bağlantı ve oturum bilgisi gösterir).
💡 İpucuKimlik doğrulama bir kereliktir; token lokal saklanır. Sonraki açılışlarda tekrar giriş gerekmez.

2.5 Model ve Effort Seçimi

/model sonnet     # doğrudan Sonnet 5'e geç
/status           # hangi model ve effort aktif, gör

2.6 VS Code Extension Kurulumu (isteğe bağlı)

  1. VS Code'u aç, Extensions görünümüne git (Cmd/Ctrl+Shift+X).
  2. 'Claude Code' ara ve kur.
  3. Command Palette (Cmd/Ctrl+Shift+P) → 'Claude Code' → panelini aç.

3.2 İzin Modları (kısa tanışma)

Claude Code'da Shift+Tab ile izin modları arasında geçersin: normal (her aksiyona onay), acceptEdits (düzenlemeleri otomatik kabul), plan (önce plan yap), auto (auto mode). Bugün normal modda kal; bu modları ve auto mode'u Gün 2'de detaylı işleyeceğiz.

3.3 İlk Kodunu Yazdır

Terminal'de Claude Code açıkken sırayla dene ve her adımda ne yaptığını gözlemle:

Deneme 1 — Basit: Python ile "Merhaba Dünya!" yazan bir script oluştur ve çalıştır. Claude bir dosya oluşturacak, kodu yazacak ve çalıştırmak için izin isteyecek.

Deneme 2 — Biraz karmaşık: Python ile bir hesap makinesi yaz; toplama/çıkarma/çarpma/bölme yapsın, kullanıcıdan input alsın, sıfıra bölmeyi handle etsin. Çalıştır ve test et.

Deneme 3 — Debugging: Bir hata mesajını yapıştırıp düzeltmesini iste (aşağıdaki kütüphanede hazır prompt var).

KÜTÜPHANE🧰 Prompt & Şablon Kütüphanesi

Kendini tanıt
Merhaba! Kendini tanıt: ne yapabilirsin, hangi araçlara sahipsin ve şu an hangi modeli kullanıyorsun?
İlk script (Hello World)
Python ile "Merhaba Dünya!" yazan bir script oluştur ve çalıştır.
Çok adımlı görev (hesap makinesi)
Python ile bir hesap makinesi yaz. Toplama, çıkarma, çarpma ve bölme yapabilsin. Kullanıcıdan input alsın. Sıfıra bölme hatasını handle etsin. Çalıştır ve test et.
Hata mesajı ile debugging
Şu hatayı alıyorum, sebebini bul ve düzelt:
Traceback (most recent call last): File "app.py", line 5, in <module> result = 10 / 0 ZeroDivisionError: division by zero
Gerçek hayatta en çok kullanacağın senaryo budur. Stack trace ne kadar tam olursa Claude o kadar iyi düzeltir.
Model karşılaştırma denemesi
Aynı görevi iki farklı effort ile dene: '/effort high' ve '/effort xhigh'. Bir bubble sort implementasyonu yazdır ve çıktı farkını gözlemle.
Effort komutu ortama göre /effort veya model seçici altında olabilir; /help ile doğrula.

CHALLENGE🏆 Günün Challenge'ı

CHALLENGE: İlk Vibe Coding Deneyimin — 'Günün Sözü' Uygulaması

Claude Code'u kullanarak her çalıştırıldığında rastgele bir motivasyon sözü gösteren bir Python uygulaması yap. Amaç kod yazmak değil; Claude'un her adımda ne yaptığını gözlemlemek.

Gereksinimler:

  • Python ile yazılsın
  • En az 10 motivasyon sözü ve yazarları olsun
  • Her çalıştırmada rastgele bir söz seçsin
  • Sözün yazarını da göstersin
  • Renkli terminal çıktısı versin (colorama veya ANSI)
  • Claude çalıştırıp test etsin

Başarı Kriterleri:

✅ Çözüm Rehberi

Tek bir net prompt ile başla, sonra iyileştir. Aşağıdaki prompt zinciri tipik bir 'yap → gözlemle → derinleştir' akışını gösterir.

Önerilen prompt zinciri:

1) İlk üretim
Python ile bir motivasyon sözü uygulaması yaz. İçinde 10 farklı motivasyon sözü ve yazarları olsun. Her çalıştırıldığında rastgele bir söz seçip renkli yazdırsın (colorama kullan). Çalıştır ve test et.
2) İyileştirme (gözlemden sonra)
Sözleri ayrı bir quotes.py dosyasına taşı, ana dosyadan import et. Ayrıca kullanıcı isterse belirli bir yazarın sözünü filtreleyebilsin (komut satırı argümanı ile).
3) Doğrulama
Uygulamayı 3 kez çalıştır ve her seferinde farklı söz geldiğini göster. Kodda eksik yorum varsa ekle.

Kontrol noktaları:

  • Claude'un hangi araçları kullandığını izle: Write (dosya), Bash (çalıştırma), Edit (iyileştirme).
  • İzin isteme anlarını gözlemle: dosya oluşturmadan ve python komutunu çalıştırmadan önce.
  • colorama kurulu değilse Claude pip install colorama çalıştırmak için izin isteyecek.

Sık yapılan hatalar:

  • ⚠️ Prompt'u fazla belirsiz bırakmak ('güzel bir uygulama yap') → net gereksinim ver.
  • ⚠️ Rastgeleliği test etmemek → en az 3 kez çalıştırıp farklı çıktı gördüğünü doğrula.
  • ⚠️ İzinleri okumadan hızlıca onaylamak → ilk günlerde her diff'i ve komutu bilinçli incele.

ÖZET📌 Takeaway'ler

  1. Claude Code bir chatbot değil, agentic bir kodlama aracıdır: dosya okur/yazar, komut çalıştırır, git yönetir.
  2. Agentic loop: Anla → Planla → Araç Kullan → Doğrula → Düzelt döngüsünde çalışır.
  3. Dört yüzey var: Terminal, IDE (VS Code/JetBrains), Desktop App ve Web. Motor hepsinde aynı.
  4. Güncel modeller: Opus 4.8, Sonnet 5 (native 1M context), Haiku 4.5 ve en yetenekli Fable 5.
  5. Effort seviyeleri (low→max) ve fast mode ile derinlik/hız dengesini ayarlarsın.
  6. Native installer önerilen yöntemdir (bağımlılık gerektirmez, otomatik güncellenir); Node.js zorunlu değil.
  7. Pro plan ($20/ay) eğitim için yeterlidir; ücretsiz plan Claude Code içermez.
  8. /status, /model, /cost, /context, /compact gibi slash komutları günlük kullanımda hayatidir.
